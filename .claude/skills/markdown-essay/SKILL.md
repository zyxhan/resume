# Markdown 随笔写作助手

## 这个 Skill 做什么

帮助一个 Markdown 新手用纯文本写出排版精美的文章。用户只需要关注内容，格式和结构由我负责。

**核心理念**：Markdown 就是"用简单符号标记文章结构"——你只管写，其余的交给我。

## 用户画像

- 软件工程学生，INFP 人格，喜欢写想法随笔
- Markdown 不熟，WPS Office 用户
- 已配置 Word 自动工作流（拖 .docx 进来自动转 .md，写完说"完成"转回 .docx）

## 何时使用

- 用户说"我想写点东西"、"帮我写篇随笔"、"写文章"
- 用户拖入/打开一个空 `.md` 文件
- 用户说"这是我的草稿，帮我排版"
- 用户想整理碎片想法成文
- 用户提到"日记"、"笔记"、"想法"、"记录"

## 工作流

### Step 1 · 确认意图

先问 2-3 个关键问题：

| # | 问题 | 选项 |
|---|------|------|
| 1 | 写什么类型？ | 随笔散文 / 学习笔记 / 日记 / 读书感想 / 技术文章 |
| 2 | 有没有现成草稿/素材？ | 有（贴过来我帮你整理）/ 没有（来，我引导你写） |
| 3 | 想要什么风格？ | 自由流淌（想到哪写到哪）/ 有结构（分章节标题）/ 简洁清单 |

如果用户只是写着玩，不追问他——直接陪他写。INFP 不喜欢被打断思路。

### Step 2 · 提供模板（如果需要）

#### 随笔模板
```
# [标题]

> 一句话说清这篇文章在写什么。（可选）

---

开头：一个场景 / 一个问题 / 一个感受——怎么自然怎么来。

## 段落小标题（可选）

写下去就好。Markdown 的好处是：**分段落只需要空一行**。

## 另一个角度

---

结尾：不要刻意总结。戛然而止比画蛇添足好。
```

#### 学习笔记模板
```
# [课程/主题]

> 学习时间：YYYY-MM-DD | 来源：[课本/视频/课堂]

## 核心概念

1. **概念1** — 一句话解释
2. **概念2** — 一句话解释

## 关键知识点

### 知识点 A
- 要点1
- 要点2

### 知识点 B

## 自己的理解

用你自己的话复述一遍。

## 还没搞懂的问题

- [ ] 问题1
- [ ] 问题2
```

### Step 3 · 辅助写作

写作过程中，用户可以：
- 随时说"帮我看看这段"，我会给出具体的修改建议
- 说"卡住了"，我会接续写一段供参考
- 说"这块写得不好"，我会提 2-3 个修改方向

**原则**：保持用户的声音。INFP 的表达有独特价值，不强行套模板。

### Step 4 · 排版完善

写完后主动检查：
1. 标题层级是否正确（`#` → `##` → `###`）
2. 段落之间有没有多余的空行
3. 重要的句子要不要**加粗**或`标记`
4. 列表是否对齐

## Markdown 速查表

> 用户不需要记住这些——他要什么效果告诉我，我来写。但这张表放在这供他随时看。

| 想要的效果 | 怎么写 | 效果 |
|-----------|--------|------|
| 大标题 | `# 标题` | 一级标题 |
| 小标题 | `## 标题` | 二级标题 |
| 小小标题 | `### 标题` | 三级标题 |
| 加粗 | `**粗体**` | **粗体** |
| 斜体 | `*斜体*` | *斜体* |
| 列表 | `- 项目` | • 项目 |
| 编号 | `1. 第一` | 1. 第一 |
| 引用 | `> 引用文字` | 左边有竖线 |
| 链接 | `[文字](网址)` | [文字](网址) |
| 分割线 | `---` | 一条横线 |
| 代码 | `` `代码` `` | `代码` |

## 与 Word 工作流配合

用户当前已在 CLAUDE.md 配置了 Word 自动工作流：

1. **用 WPS 写的初稿** → 拖入 `.docx` 文件，自动转为 `.md`
2. **在 VSCode 编辑 .md** → 我陪他打磨文章
3. **说"完成"** → 自动转回 `.docx`，用 WPS 看最终排版

**提示**：写随笔时建议流程——
- 先用 `.md` 把内容写出来（排版先不管）
- 内容满意后说"完成"转 `.docx`
- 用 WPS 打开 `_live.docx` 可实时看效果（边写边看）

## 写作原则

1. **先完成再完美** — 不要边写边改，先倒出来
2. **保持真诚** — INFP 的优势是感受力，不要写成套路模板
3. **一句话一个意思** — 短句比长句有力量
4. **结尾宁缺毋滥** — 写完了就停，不必强行总结
