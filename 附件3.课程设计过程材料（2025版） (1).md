## 设计方案

本系统采用 **C 语言（C99 标准）** 开发，运行于 Windows CLI 命令行终端，基于**四层架构**（表现层、业务层、数据层、持久化层）组织，划分为六大功能模块。

**（1）系统架构设计**

系统自上而下分为四层：表现层（CLI 菜单与格式化输出界面）、业务层（六大功能模块）、数据层（内存数据结构与索引）、持久化层（二进制文件序列化）。各模块通过 `SystemState` 全局状态单例共享图书数组、读者数组、借阅记录链表、ISBN 哈希索引和分类索引，模块间低耦合、高内聚。

**（2）数据结构选型**

引入 klib 轻量级 C 库（零外部依赖），选用动态数组（`kvec`）存储图书和读者列表，链表（`klist`）管理借阅记录，哈希表（`khash`）建立 ISBN→数组下标 和 读者ID→数组下标 的 O(1) 快速索引。系统支持数据规模：图书 ≤ 20,000 种、读者 ≤ 10,000 人、借阅记录 ≤ 50,000 条。

**（3）智能检索模块（模块5，本人负责）**

实现大小写不敏感的模糊子串匹配算法，支持对书名、作者、ISBN、出版社、分类五个字段的 AND 组合检索；提供分类树浏览与分页展示（每页可配置）；支持按书名、作者、借阅热度、出版日期多维度升降序排序；基于累计借阅次数实现热门图书 Top-N 推荐。检索召回率目标 ≥ 90%，单次检索响应 ≤ 1 秒（万条数据）。

**（4）其他功能模块概要**

模块1（读者分类管理）：支持本科生、研究生、教师、职工四种身份注册，每种身份对应差异化借阅权限（最大可借数、借阅天数、续借次数）。模块2（图书借还管理）：图书增删改查、多副本库存管理、借阅资格自动校验（逾期/欠费/借满拦截）、归还与续借。模块3（逾期罚金管理）：自动逾期检测、按天罚金计算、催还通知、罚金缴纳与豁免。模块4（统计报表展示）：借阅排行榜（日/周/月/年）、分类统计、读者活跃度分析、文本可视化图表。模块6（数据持久化）：全量保存/加载、定时自动备份、手动备份/恢复、操作日志记录、系统配置管理。

**（5）界面与交互设计**

使用 ANSI 24bit 真彩色 + Unicode 双线框绘制 CLI 界面，设计 Neon Archive 主题配色。检索结果以格式化表格呈现（序号/ISBN/书名/作者/出版社/分类/可借数/热度），隔行变色，可借状态绿色、无库存红色标识。

**（6）构建与运行环境**

编译器采用 MinGW GCC（`gcc -std=c99 -O2`），构建工具为 GNU Make，运行时需设置控制台代码页为 UTF-8（`chcp 65001`）。可执行文件为 `bin/library.exe`，数据文件存储于 `bin/data/` 目录。

## 设计要求

**（1）读者分类管理模块：** 实现本科生、研究生、教师、职工四种身份的注册、查询、状态管理（正常/挂失/注销）和类型变更功能。每种身份配置差异化借阅权限：本科生最大可借 5 本/30 天，研究生 10 本/60 天，教师 15 本/90 天，职工 5 本/30 天。权限验证准确率要求达到 100%。

**（2）图书借还管理模块：** 实现图书的增删改查、多副本库存管理（总副本数/可用副本数）。借阅操作自动校验：读者是否存在、是否挂失/注销、是否借满、是否有逾期未还、是否有未缴罚金。归还操作自动计算是否逾期并触发罚金流程。续借操作限制次数（本科生 1 次、研究生 2 次、教师 3 次）。单次操作响应时间 ≤ 1.5 秒。

**（3）逾期罚金管理模块：** 自动检测所有借阅记录中的逾期项，按 0.1 元/本/天 计算罚金，精确到分。支持查询逾期列表、催还通知打印、罚金缴纳和特殊情况罚金豁免。系统配置中可设置宽限期（默认 3 天）。

**（4）统计报表展示模块：** 提供借阅热度排行榜（日榜/周榜/月榜/年榜 TOP 20）、分类图书统计（各分类总量/在借量/可借量）、读者活跃度排名、逾期率统计。支持文本字符柱状图可视化。报表生成时间 ≤ 3 秒。

**（5）智能检索系统模块：** 实现基础模糊检索（书名/作者关键字）、高级组合检索（书名+作者+分类+出版社 AND 逻辑）、分类浏览（分页）、多维度排序（相关性/热度/出版年份/书名/作者）和热门图书 Top-N 推荐。检索召回率 ≥ 90%，精确检索准确率 100%。

**（6）数据持久化模块：** 支持全量数据保存/加载、退出时自动保存、定时自动备份（可配置间隔）、手动备份/恢复至指定目录、操作日志记录与查询、系统配置（罚金费率/宽限期/备份策略等）的动态修改与持久化。数据持久化成功率要求 ≥ 99.99%。

## 工作量

**（1）代码规模：** 系统总计约 5,000 行 C 代码，包含 10 个 .c 源文件（main.c / common.c / data_types.c / menu_cli.c / module1_reader.c / module2_borrow.c / module3_overdue.c / module4_stats.c / module5_search.c / module6_persist.c）、5 个 .h 头文件、4 个 klib 库头文件（kvec.h / klist.h / khash.h / ksort.h）。

**（2）模块5 工作量（本人负责）：** 约 506 行 C 代码，包含 7 个对外接口（基础检索/高级检索/分类浏览/结果排序/热门推荐/分页显示/图书详情），5 个内部辅助函数（模糊匹配/结果表格输出/4 种排序比较器），35 本演示图书数据覆盖计算机/文学/数学/历史/哲学/经济/心理 8 个分类。

**（3）接口总量：** 系统共定义 44 个对外接口，其中读者管理 8 个、图书管理 6 个、借还操作 6 个、逾期罚金 6 个、检索 7 个、报表 5 个、持久化 8 个。

**（4）测试规模：** 38 项验收测试用例，覆盖六大模块的核心功能路径和边界条件。自动化测试脚本 `run_test.ps1` 支持批量回归测试。

## 工作计划

本系统由五名成员分工协作完成，各模块分配如下：

李媛心负责模块一（读者分类管理）和模块二（图书借还管理）：实现本科生、研究生、教师、职工四种身份的注册、查询、状态管理、类型变更与差异化权限配置；实现图书增删改查、多副本库存管理、借阅/归还/续借及资格自动校验（挂失/注销/借满/逾期/欠费五项检查）。

韦爱佳负责模块三（逾期罚金管理）：自动检测全部借阅记录中的逾期项，按0.1元/本/天计算罚金（精确到分），支持宽限期配置（默认3天）、催还通知、罚金缴纳与特殊情况豁免。

陈宇蝶负责模块四（统计报表展示）：提供借阅热度排行榜（日/周/月/年TOP20）、分类图书统计（各分类总量/在借量/可借量）、读者活跃度排名、逾期率分析，支持文本字符柱状图可视化呈现。

朱轶涵负责模块五（智能检索系统）：实现大小写不敏感的模糊子串匹配算法，支持书名、作者、ISBN、出版社、分类五字段AND组合检索，提供分类树浏览与分页展示，支持按书名、作者、热度、出版日期多维度升降序排序，基于累计借阅次数实现热门图书Top-N推荐。

郭传鹏负责模块六（数据持久化与系统管理）：全量数据二进制序列化保存/加载、退出自动保存、定时自动备份、手动备份/恢复至指定目录、操作日志记录与查询、系统配置（罚金费率/宽限期/备份策略）的动态管理。

各成员独立完成所负责模块的编码与单元测试，全员参与需求分析与系统概要设计。最后一周进行系统集成联调、性能优化、边界条件验证、CLI界面美化与课程设计报告撰写。
