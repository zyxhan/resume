## Task 1：STM32入门（第1-3周）【必做】

### 开发环境搭建
- 安装 **Keil MDK**（正点原子/野火教程里有安装包）
- 安装 **STM32CubeMX**（ST官方的图形化配置工具）
- 连接ST-Link，烧录第一个LED闪烁程序

### 学习内容（跟着视频做）

**GPIO**
- 点亮LED（输出）
- 按键检测（输入）
- 按键消抖（软件消抖）

**UART串口通信**
- `printf`重定向到串口（用串口助手看输出）
- 串口收发数据
- 通过串口发送指令控制LED

**定时器**
- 基本定时器中断
- PWM输出（控制LED亮度/舵机）

**中断**
- 外部中断（按键触发）
- 中断优先级概念

**I2C通信**
- I2C协议原理（起始信号、地址、读写、应答）
- 驱动OLED屏显示文字和数据

**SPI通信**
- SPI协议原理（与I2C的区别）
- 了解即可，不一定要做项目

### 推荐资源
- 视频：B站正点原子STM32入门教程（免费）
- 文档：正点原子《STM32开发指南》PDF（免费下载）

### 项目1：串口调试助手
```
功能：
├── 串口发送指令控制LED开关
├── 串口打印传感器数据
├── 支持简单的指令解析（如"LED ON"、"LED OFF"）
└── 波特率可配置
```

### 项目2：温湿度监测系统
```
功能：
├── DHT11定时采集温湿度
├── OLED屏幕实时显示
├── 串口上报数据到PC
├── 温度超阈值报警（蜂鸣器或LED闪烁）
└── 用到：GPIO + I2C + UART + 定时器
```

## Task 2：FreeRTOS入门（第4-5周）【尽量做】

### 为什么要学RTOS？
- 字节面试明确考了"FreeRTOS任务通知原理"
- 嵌入式岗位几乎都要求了解RTOS
- 有了STM32基础，学FreeRTOS只需要1-2周

### 核心概念
- **任务（Task）**：创建、删除、优先级
- **任务调度**：抢占式调度原理
- **信号量（Semaphore）**：任务同步
- **队列（Queue）**：任务间传递数据
- **互斥锁（Mutex）**：保护共享资源
- **任务通知**：轻量级的事件通知机制（字节考点）

### 推荐资源
- 视频：B站韦东山FreeRTOS教程（免费）
- 文档：野火《FreeRTOS源码详解与应用开发》PDF

### 项目：多任务温湿度系统
```
架构（3个任务）：
├── 采集任务：定时读取DHT11数据，写入队列
├── 显示任务：从队列读数据，刷新OLED显示
└── 报警任务：监测温度，超阈值触发蜂鸣器

用到：Task + Queue + Semaphore + 定时器
```

## 硬件调试心态指南

调硬件和纯软件不一样，要有心理准备：
- **接线错误**是90%的问题来源，先检查接线再看代码
- **供电不足**会导致各种奇怪现象，USB供电要够
- **杜邦线接触不良**很常见，换一根线试试
- **别急**，每个问题都是学习机会
