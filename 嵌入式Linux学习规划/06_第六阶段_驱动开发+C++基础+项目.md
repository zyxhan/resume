## Task 1：Linux驱动入门（9月-11月）【必做】

### 驱动开发是什么？

```
应用层程序 → 系统调用 → [驱动层] → 硬件
```

驱动就是应用和硬件之间的桥梁。你写的LED控制程序通过驱动去操作GPIO寄存器。

### 核心知识点

**字符设备驱动框架**
- 设备号（主设备号 + 次设备号）
- `cdev`结构体注册
- `file_operations`（open/read/write/ioctl实现）
- 用户空间与内核空间数据传递（`copy_to_user`/`copy_from_user`）

**Platform总线模型**
- 为什么要用Platform总线？（驱动与设备分离）
- Platform驱动和设备的注册
- 匹配机制

**子系统驱动**
- GPIO子系统（`gpiod` API）
- I2C子系统驱动框架
- Input子系统（按键/触摸屏输入）

**中断处理**
- 顶半部（中断处理函数，快速响应）
- 底半部（tasklet/workqueue，耗时处理）
- 字节面经考点："中断分段"原理和原因

### 推荐资源
- 正点原子嵌入式Linux驱动教程
- 书：《Linux设备驱动程序》（经典但较老，选读）

### 项目1：按键驱动（Input子系统）
```
功能：
├── 注册Input设备
├── 按键按下/释放事件上报

├── 应用层通过 /dev/input/eventX 读取按键事件
└── 用到：中断 + Input子系统 + 底半部
```

### 项目2：I2C温湿度传感器驱动
```
功能：
├── 基于I2C子系统框架编写驱动
├── 读取温湿度数据

```

**字节BSP驱动岗相关考点**：
- 熟悉Linux内核源码：进程调度、内存管理、文件系统、设备子系统
- 熟悉UART/I2C/SPI通信协议
- 有示波器、逻辑分析仪基础操作能力

## Task 2：C++基础（11月-12月，约2周）【尽量做】

### 为什么嵌入式岗要学C++？
- 字节面试明确考了"虚指针虚表原理"
- 很多嵌入式项目是C/C++混编
- 不需要深入，理解核心概念即可

### 核心知识点（2周速成）

**第1周：面向对象基础**
- 类与对象、构造函数/析构函数
- 封装（public/private/protected）
- 继承（public继承）
- 多态（虚函数、纯虚函数、抽象类）

**第2周：深入 + STL**
- 虚函数表原理（vptr和vtable）—— 字节面试考点
- 构造函数调用顺序
- STL基础：`vector`、`map`、`string`、`algorithm`
- 智能指针概念（`unique_ptr`、`shared_ptr`）

### 推荐资源
- B站：搜"C++面向对象"，找一个20小时以内的速成课
- 不需要做大型项目，理解概念为主

### 学习方法
- 用C++重写一个小项目（比如把通讯录用C++类重新实现）
- 重点是能**解释**虚函数表原理，画图说明

## Task 3：综合项目（12月-1月）【必做】选1个

### 项目A：智能家居网关（强烈推荐）

```
架构：
┌─────────────┐    MQTT    ┌──────────────┐    HTTP    ┌─────────┐
│   STM32     │ ─────────→ │   IMX6ULL    │ ─────────→ │  手机   │
│  传感器采集  │   WiFi     │  Linux网关   │            │  查看   │
└─────────────┘            └──────────────┘            └─────────┘

技术栈：
├── STM32端：FreeRTOS + 传感器 + MQTT客户端
├── IMX6ULL端：MQTT Broker + 数据处理 + HTTP API
├── 驱动：I2C传感器驱动 + GPIO控制
└── 通信：MQTT + HTTP + Socket

这个项目覆盖了：
- 嵌入式Linux（IMX6ULL）
- 单片机（STM32 + FreeRTOS）
- 网络编程（MQTT + HTTP）
- 驱动开发（I2C/GPIO）
- 数据结构（数据处理）
```

### 项目B：智能农业监测站

```
功能：

```
