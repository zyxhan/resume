# 第五阶段：嵌入式Linux核心

> **时间**：大二暑假（2027年7月-8月），共8周
> **每天投入**：4-6小时
> **目标**：打通嵌入式Linux全链路（U-Boot → 内核 → 文件系统 → 应用）
> **前置要求**：完成第四阶段

---

## 这个阶段在做什么？

简单说就是：**从零构建一个完整的嵌入式Linux系统**

```
开发板上电 → U-Boot启动 → 加载Linux内核 → 挂载根文件系统 → 运行应用程序
```

走完这个流程，你就真正理解了嵌入式设备是怎么跑起来的。

---

## 总体安排

| 周次 | 内容 | 优先级 |
|------|------|--------|
| 第1-2周 | 交叉编译 + U-Boot | 必做 |
| 第3-4周 | 内核编译 + 设备树 | 必做 |

## Task 1：交叉编译 + U-Boot（第1-2周）【必做】

### 交叉编译
- 什么是交叉编译？（在PC上编译，在开发板上运行）
- 搭建交叉编译工具链（arm-linux-gnueabihf-gcc）
- 编译一个Hello World在开发板上运行

### U-Boot
- U-Boot是什么？（引导加载程序，相当于嵌入式设备的BIOS）
- 下载源码，配置并编译（`make imx6ull_defconfig`）
- 烧录到开发板（SD卡/eMMC）
- U-Boot常用命令：`printenv`、`boot`、`md`、`mw`
- 环境变量配置（bootcmd、bootargs）

### 学习重点
- 理解启动流程：上电 → iROM → U-Boot → 内核
- 学会修改U-Boot环境变量来改变启动参数

---

## Task 2：内核编译 + 设备树（第3-4周）【必做】

### Linux内核编译
- 下载内核源码（正点原子提供的版本）
- `make menuconfig`（图形化内核配置）
- 编译内核镜像（zImage）
- 烧录并在开发板启动

### 设备树（Device Tree）
- 设备树是什么？（描述硬件信息的数据结构）
- DTS语法基础（节点、属性、标签）
- 修改设备树：添加一个LED节点
- `dtc`编译工具，生成DTB文件

### 推荐资源
- 正点原子《I.MX6ULL嵌入式Linux开发指南》
- 视频：正点原子嵌入式Linux视频教程

---

## Task 3：根文件系统（第5-6周）【必做】

### 根文件系统是什么？
- Linux启动后挂载的第一个文件系统（`/`）
- 包含 `/bin`、`/etc`、`/lib`、`/dev` 等目录
- 没有它，内核启动了也没法用

### 使用Buildroot构建
- 下载并配置Buildroot
- 选择需要的软件包（busybox、ssh、网络工具等）
- 编译生成根文件系统镜像
- 烧录到开发板

### 手动构建（理解原理）
- 手动创建目录结构
- 移植busybox
- 配置inittab和fstab
- 挂载proc/sys文件系统

---

## Task 4：应用层开发（第7-8周）【必做】

### 在开发板上运行你的程序
- 交叉编译你在Linux下写的应用程序
- 通过NFS/TFTP/SCP传输到开发板
- 在开发板上运行和调试

### 项目：开发板LED控制程序
```
功能：
├── 通过设备树配置LED GPIO
├── 应用层通过 /sys/class/leds 控制LED
├── 实现闪烁、呼吸灯效果
├── 提供简单的命令行交互界面
└── 综合了：设备树 + 应用层编程 + 调试
```

### 完整系统验证
- 从U-Boot到应用层，完整走一遍启动流程
- 确认每一步都在做什么
- 记录整个流程（这就是你的面试素材）
