# 模块6：数据持久化与系统管理 PRD

## 6.1 需求概述

### 6.1.1 背景
图书馆系统需要长期稳定运行，数据安全与系统可维护性是基础保障。需实现数据的可靠存储、备份恢复和系统配置管理。

### 6.1.2 目标
- 实现数据的可靠持久化（无数据库依赖）
- 提供自动备份与手动备份机制
- 支持数据恢复与迁移
- 提供系统配置与日志管理

### 6.1.3 范围

| 包含 | 不包含 |
|------|--------|
| 二进制文件序列化 | 数据库存储 |
| 自动/手动备份 | 云存储备份 |
| 数据恢复 | 分布式数据同步 |
| 系统配置管理 | 网络配置管理 |

## 6.3 详细功能需求

### 6.3.1 数据序列化（FR-601 ~ FR-610）

#### FR-601 序列化文件清单

| 文件名 | 数据内容 | 结构 | 大小估算 |
|--------|---------|------|---------|
| `books.dat` | 图书数组 | [count][Book×count] | ~2MB/万种 |
| `readers.dat` | 读者数组 | [count][Reader×count] | ~1MB/万人 |
| `records.dat` | 借阅记录链表 | [count][Record×count] | ~3MB/5万条 |
| `config.ini` | 系统配置 | 文本键值对 | ~1KB |
| `library.log` | 运行日志 | 文本行 | 动态增长 |

#### FR-602 图书数据序列化格式

```
┌─────────┬─────────┬─────────┬─────────┐
│  int    │  Book   │  Book   │  ...    │
│ count   │  [0]    │  [1]    │         │
│ 4字节   │ sizeof  │ sizeof  │         │
│         │ (Book)  │ (Book)  │         │
└─────────┴─────────┴─────────┴─────────┘
```

#### FR-603 记录数据序列化格式（链表）

```
┌─────────┬─────────┬─────────┬─────────┐
│  int    │ Record  │ Record  │  ...    │
│ count   │ node 1  │ node 2  │         │
│ 4字节   │ (不含   │ (不含   │         │
│         │ next指针)│ next指针)│        │
└─────────┴─────────┴─────────┴─────────┘
```

**加载时重建链表**：
```c
RecordNode *head = NULL, *tail = NULL;
for (int i = 0; i < count; i++) {
    RecordNode *node = malloc(sizeof(RecordNode));
    fread(node, sizeof(RecordNode) - sizeof(void*), 1, fp);
    node->next = NULL;
    if (!head) head = tail = node;
    else { tail->next = node; tail = node; }
}
```

### 6.3.2 数据加载（FR-611 ~ FR-620）

#### FR-611 启动加载流程

```
检测数据文件存在性
    │
    ├─ 全部存在 ──► 逐个加载 ──► 校验完整性
    │                              │
    │                              ├─ 通过 ──► 正常启动
    │                              │
    │                              └─ 失败 ──► 进入恢复流程
    │
    ├─ 部分缺失 ──► 提示缺失文件 ──► 尝试恢复或初始化
    │
    └─ 全部缺失 ──► 首次运行 ──► 提示创建演示数据
                                   │
                                   ├─ 是 ──► 加载演示数据
                                   │
                                   └─ 否 ──► 创建空库
```

#### FR-612 数据完整性校验

| 校验项 | 校验方法 | 修复策略 |
|--------|---------|---------|
| 文件头魔数 | 固定字节序列 `LIB\x00` | 无法修复，需恢复 |
| 记录数量一致性 | 文件头count vs 实际记录 | 以实际记录为准 |
| 数据CRC校验 | 循环冗余校验 | 尝试修复，失败则恢复 |
| 关联一致性 | 借阅记录中的reader_id/book_id是否存在 | 标记孤儿记录 |
| 数值范围 | 数量、金额是否在合理范围 | 标记异常，人工审核 |

### 6.3.3 备份管理（FR-621 ~ FR-630）

#### FR-621 自动备份策略

| 触发条件 | 备份文件命名 | 保留数量 |
|---------|------------|---------|
| 每日首次修改 | `backup_YYYYMMDD_0200.dat` | 最近7天 |
| 批量导入后 | `backup_YYYYMMDD_HHMM.dat` | 最近3次 |
| 手动触发 | `backup_manual_YYYYMMDD_HHMM.dat` | 最近10次 |

#### FR-622 备份文件结构

```
backup_20260606_0200.dat
├── 头部标识: "LIBBACKUP\x00"
├── 备份时间戳: time_t
├── 备份类型: 0=自动, 1=手动
├── 数据块1: 图书数据
├── 数据块2: 读者数据
├── 数据块3: 记录数据
├── 配置块: 系统配置
└── 校验和: CRC32
```

#### FR-623 备份清理策略

- 自动备份：保留最近7天，超期自动删除
- 手动备份：保留最近10个，超期需人工确认
- 磁盘空间<<100MB时：仅保留最近3个备份

### 6.3.4 恢复管理（FR-631 ~ FR-640）

#### FR-631 恢复触发条件

| 触发条件 | 处理方式 | 自动/手动 |
|---------|---------|----------|
| 启动时数据文件损坏 | 自动尝试恢复最近备份 | 自动 |
| 误删除数据 | 管理员选择备份点恢复 | 手动 |
| 数据不一致 | 校验失败，提示修复或恢复 | 手动 |
| 系统迁移 | 导入旧系统备份文件 | 手动 |

#### FR-632 恢复流程

```
检测到数据异常
    │
    ▼
显示异常信息，提供选项：
    │
    ├── [1] 尝试自动修复 ──► 修复校验和，重建索引
    │                         │
    │                         ├─ 成功 ──► 正常启动
    │                         │
    │                         └─ 失败 ──► 进入恢复流程
    │
    ├── [2] 从备份恢复 ──► 列出可用备份点
    │                         │
    │                         ▼
    │                   选择备份文件
    │                         │
    │                         ▼
    │                   确认恢复（警告：当前数据将丢失）
    │                         │
    │                         ▼
    │                   执行恢复，验证完整性
    │                         │
    │                         ├─ 成功 ──► 正常启动
    │                         │
    │                         └─ 失败 ──► 尝试更早备份
    │
    └── [3] 初始化新系统 ──► 清空数据，创建空库
                              │
                              ▼
                        进入首次运行模式
```

#### FR-633 备份点列表格式

```
可用备份点：
────────────────────────────────────────
  序号  文件名                    时间              大小    状态
  ────────────────────────────────────────────────
  1     backup_20260606_0200.dat  2026-06-06 02:00  2.3MB   ✓ 完整
  2     backup_20260605_0200.dat  2026-06-05 02:00  2.2MB   ✓ 完整
  3     backup_manual_20260604.dat 2026-06-04 15:30 2.1MB   ✓ 完整
  4     backup_20260604_0200.dat  2026-06-04 02:00  2.1MB   ⚠ 部分
────────────────────────────────────────
  请输入序号选择备份点 (0取消):
```

### 6.3.5 系统配置（FR-641 ~ FR-650）

#### FR-641 可配置参数

| 参数名称 | 默认值 | 范围 | 说明 |
|---------|--------|------|------|
| `auto_backup` | 1 | 0/1 | 是否启用自动备份 |
| `backup_interval` | 86400 | 3600-604800 | 自动备份间隔（秒） |
| `backup_keep_days` | 7 | 1-30 | 自动备份保留天数 |
| `max_books_undergrad` | 5 | 1-20 | 本科生最大可借数 |
| `max_days_undergrad` | 30 | 7-180 | 本科生借阅天数 |
| `fine_rate` | 0.5 | 0.1-5.0 | 罚金费率（元/天） |
| `max_renew` | 2 | 0-5 | 最大续借次数 |
| `grace_period` | 1 | 0-7 | 宽限期（天） |
| `session_timeout` | 1800 | 300-7200 | 会话超时（秒） |

#### FR-642 配置文件格式

```c
/* config.ini - 文本配置文件 */
#define CONFIG_FILE "library.conf"

typedef struct {
    int auto_backup;            /* 自动备份开关 */
    int backup_interval;        /* 备份间隔 */
    int backup_keep_days;       /* 保留天数 */
    int max_books[4];           /* 各类型最大可借数 */
    int max_days[4];            /* 各类型借阅天数 */
    float fine_rate;            /* 罚金费率 */
    int max_renew;              /* 最大续借次数 */
    int grace_period;           /* 宽限期 */
    int session_timeout;        /* 会话超时 */
    char admin_password[32];    /* 管理员密码 */
    char library_name[64];      /* 图书馆名称 */
} SystemConfig;
```

## 6.7 验收标准

| 测试项 | 步骤 | 预期结果 |
|-------|------|---------|
| 数据保存 | 添加100条记录后退出 | 文件大小增加，数据完整 |
| 自动备份 | 等待24小时或修改触发 | 生成新备份文件 |
