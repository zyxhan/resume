# 模块2：图书借还管理 PRD

## 2.1 需求概述

### 2.1.1 背景
图书借还是图书馆最核心的业务环节，需要确保流程高效、准确，同时支持多副本管理和实时库存更新。

### 2.1.2 目标
- 实现标准化的借阅/归还/续借流程
- 支持多副本图书的精确库存管理
- 借阅操作与读者权限实时联动校验
- 操作记录可追溯，支持历史查询

### 2.1.3 范围

| 包含 | 不包含 |
|------|--------|
| 借阅、归还、续借操作 | 馆际互借 |
| 多副本库存管理 | RFID自助借还设备对接 |
| 借阅历史记录 | 图书预约排队系统 |

## 2.3 详细功能需求

### 2.3.1 借阅操作（FR-101 ~ FR-110）

#### FR-101 借阅权限校验链

校验顺序（任一失败则终止）：

```
1. 读者登录状态校验 ──► 2. 账户状态校验 ──► 3. 借阅上限校验
       │                      │                      │
       ▼                      ▼                      ▼
   未登录→拒绝          冻结/过期→拒绝         当前借阅≥max→拒绝
       │                      │                      │
       └──────────────────────┴──────────────────────┘
                              │
                              ▼
                    4. 欠费校验 ──► 5. 图书可借校验
                       │                │
                       ▼                ▼
                  欠费>0→拒绝      可用副本=0→拒绝
                       │                │
                       └────────────────┘
                              │
                              ▼
                         6. 同书重复校验
                              │
                              ▼
                        已借同书→拒绝
                              │
                              ▼
                         ✅ 校验通过，执行借阅
```

#### FR-102 借阅流程

| 步骤 | 操作 | 数据更新 |
|-----|------|---------|
| 1 | 输入图书编号 | 查询图书信息 |
| 2 | 确认借阅 | 减少可用副本数 |
| 3 | 生成借阅记录 | 写入记录链表 |
| 4 | 更新读者状态 | 已借数+1 |
| 5 | 更新图书热度 | 借阅次数+1 |
| 6 | 保存数据 | 持久化到文件 |

#### FR-103 借阅记录生成

```c
typedef struct RecordNode {
    int record_id;              /* 记录编号（唯一） */
    int book_id;                /* 图书编号 */
    int reader_id;              /* 读者编号 */
    int copy_id;                /* 副本编号（多副本管理） */
    time_t borrow_time;         /* 借阅时间 */
    time_t due_time;            /* 应还时间 = borrow_time + max_days * 86400 */
    time_t return_time;         /* 实际归还时间（0表示未归还） */
    int renew_count;            /* 已续借次数 */
    float fine_amount;          /* 罚金金额 */
    RecordStatus status;        /* 记录状态 */
    struct RecordNode *next;    /* 链表指针 */
} RecordNode;
```

**应还时间计算**：
```c
due_time = borrow_time + (reader->max_days * 86400);
/* 若到期日为节假日，自动顺延至下一个工作日（简化版：不加判断） */
```

### 2.3.2 归还操作（FR-111 ~ FR-120）

#### FR-111 归还流程

```
输入借阅记录编号
    │
    ▼
查询记录 ──► 不存在/已归还 ──► 错误提示
    │
    ▼
确认归还
    │
    ▼
计算逾期天数 = 当前时间 - 应还时间
    │
    ├─ 逾期天数 ≤ 0 ──► 状态=已归还，无罚金
    │
    └─ 逾期天数 > 0 ──► 罚金 = 逾期天数 × 0.5元
              │
              ▼
        状态=已逾期，更新读者欠费
              │
              ▼
        释放库存（可用副本+1）
              │
              ▼
        保存数据
```

#### FR-112 罚金计算规则

| 逾期天数 | 罚金单价 | 日上限 | 累计上限 |
|---------|---------|--------|---------|
| 1-30天 | 0.5元/天 | 0.5元 | 15元 |
| 31-60天 | 0.8元/天 | 0.8元 | 39元 |
| 61-90天 | 1.0元/天 | 1.0元 | 69元 |
| >90天 | 图书原价 | - | 图书原价 |

**简化实现**（当前版本）：统一 0.5元/天，无上限

#### FR-113 批量归还

- 支持一次输入多个记录编号，逗号分隔
- 批量校验所有记录合法性
- 任一记录非法则全部回滚
- 全部合法则统一执行

### 2.3.3 续借操作（FR-121 ~ FR-130）

#### FR-121 续借资格校验

| 校验项 | 规则 | 失败提示 |
|-------|------|---------|
| 记录状态 | 必须为"借阅中"或"已续借" | "该记录不可续借" |
| 续借次数 | < 最大续借次数 | "已达最大续借次数" |
| 逾期状态 | 当前时间 ≤ 应还时间 | "已逾期，无法续借" |
| 预约冲突 | 无其他读者预约该书 | "该书已被预约，无法续借" |
| 欠费状态 | 读者无欠费 | "存在欠费，无法续借" |

#### FR-122 续借期限计算

```
新应还时间 = 原应还时间 + (读者类型对应借阅天数 * 86400)
/* 或 */
新应还时间 = 当前时间 + (读者类型对应借阅天数 * 86400)
```

**策略选择**：以原应还时间为基准延长，避免读者"钻空子"临近到期续借

---

### 2.3.4 库存管理（FR-131 ~ FR-140）

#### FR-131 多副本状态模型

| 状态 | 含义 | 可借 | 位置 |
|------|------|------|------|
| 在架 | 在图书馆书架上 | ✓ | 馆藏位置 |
| 借出 | 被读者借阅 | ✗ | 读者持有 |
| 预约 | 被读者预约锁定 | ✗ | 保留架 |
| 损坏 | 待修复 | ✗ | 修复区 |
| 遗失 | 已报失 | ✗ | - |
| 编目中 | 新书上架处理中 | ✗ | 采编部 |

#### FR-132 库存数据结构

```c
typedef struct {
    int id;                     /* 图书编号（品种级） */
    char isbn[20];              /* ISBN */
    char title[100];            /* 书名 */
    char author[50];            /* 作者 */
    char publisher[50];         /* 出版社 */
    int year;                   /* 出版年份 */
    char category[30];          /* 分类 */
    char location[30];          /* 馆藏位置 */
    float price;                /* 单价 */
    int total_copies;           /* 总副本数 */
    int available;              /* 可借副本数 */
    int borrowed_count;         /* 累计借阅次数（热度指标） */
    BookStatus status;          /* 品种状态 */
} Book;
```

### 2.3.5 历史记录（FR-141 ~ FR-150）

#### FR-141 个人借阅历史

| 字段 | 说明 |
|------|------|
| 记录编号 | 唯一标识 |
| 图书信息 | 书名、作者、ISBN |
| 借阅时间 | 精确到秒 |
| 应还时间 | 精确到天 |
| 归还时间 | 精确到秒（未归还显示"-"） |
| 续借次数 | 0-3次 |
| 状态 | 借阅中/已归还/已逾期/已续借 |
| 罚金 | 金额或"-" |

#### FR-142 排序与筛选

- 默认按借阅时间倒序
- 支持按状态筛选（只看逾期、只看已归还）
- 支持按图书分类筛选

---

### 2.3.6 异常处理（FR-151 ~ FR-160）

#### FR-151 损坏登记

| 损坏等级 | 赔偿比例 | 处理 |
|---------|---------|------|
| 轻微（涂画、折页） | 原价10% | 修复后继续流通 |
| 中等（缺页、水渍） | 原价30% | 修复后降级为参考阅览 |
| 严重（散页、撕裂） | 原价50% | 下架处理 |
| 完全损毁 | 原价100% | 注销，重新采购 |

#### FR-152 遗失赔偿

```
赔偿金额 = 图书原价 × 赔偿倍数
赔偿倍数 = 2.0（1年内出版）
         = 3.0（1-3年出版）
         = 5.0（3年以上出版/绝版）
```

赔偿后，读者可保留该书或归还图书馆（由图书馆决定）

## 2.4 非功能需求

### 2.4.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 单次借阅操作响应 | ≤1秒 |
| 单次归还操作响应 | ≤1.5秒（含罚金计算） |
| 批量归还（10本） | ≤3秒 |
| 历史记录查询（1000条） | ≤2秒 |

### 2.4.2 并发处理

- 当前版本为单用户CLI，暂不考虑并发
- 预留锁机制接口：`lock_book(int book_id)`, `unlock_book(int book_id)`

## 2.7 验收标准

| 测试项 | 步骤 | 预期结果 |
|-------|------|---------|
| 正常借阅 | 合法读者借可借图书 | 生成记录，库存-1 |
| 权限不足 | 已借5本的本科生借第6本 | 拒绝，提示已达上限 |
| 欠费借阅 | 欠费读者尝试借阅 | 拒绝，提示先缴费 |
| 逾期归还 | 逾期5天归还 | 计算罚金2.5元 |
| 正常续借 | 未逾期记录续借 | 应还时间延长30天 |
| 逾期续借 | 已逾期记录续借 | 拒绝，提示先归还 |
| 超次续借 | 已续借2次再次续借 | 拒绝，提示已达上限 |
