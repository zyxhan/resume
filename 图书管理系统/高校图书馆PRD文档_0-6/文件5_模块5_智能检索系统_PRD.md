# 模块5：智能检索系统 PRD

## 5.1 需求概述

### 5.1.1 背景
图书馆馆藏数万册图书，读者需要快速、精准地找到所需资源。传统检索仅支持单一字段，无法满足复杂查询需求。

### 5.1.2 目标
- 支持多维度组合检索
- 实现模糊匹配与智能提示
- 提供分类浏览与热门推荐
- 检索结果可按相关性/热度/时间排序

### 5.1.3 范围

| 包含 | 不包含 |
|------|--------|
| 多字段组合检索 | 自然语言语义理解 |
| 模糊匹配与通配符 | 图像识别检索 |
| 分类索引浏览 | 跨库 federated search |
| 检索历史记录 | 个性化推荐算法 |

## 5.3 详细功能需求

### 5.3.1 基础检索（FR-401 ~ FR-410）

#### FR-401 检索字段

| 字段 | 匹配方式 | 示例 |
|------|---------|------|
| 书名 | 模糊匹配（包含） | "程序" → "C程序设计"、"Java程序设计" |
| 作者 | 模糊匹配 | "严" → "严蔚敏" |
| ISBN | 精确/前缀匹配 | "978-7-111" |
| 出版社 | 模糊匹配 | "机械工业" |
| 分类 | 精确匹配 | "计算机" |
| 出版年份 | 范围匹配 | "2020-2022" |

#### FR-402 模糊匹配算法

```c
/* 简化实现：子串匹配 */
int fuzzy_match(const char *text, const char *keyword) {
    return strstr(text, keyword) != NULL;
}

/* 支持大小写不敏感 */
int fuzzy_match_ci(const char *text, const char *keyword) {
    /* 转换为大写后比较 */
}
```

#### FR-403 通配符支持

| 通配符 | 含义 | 示例 |
|--------|------|------|
| `*` | 任意字符序列（0-N） | `*程序*` 匹配所有含"程序" |
| `?` | 单个任意字符 | `C?程序` 匹配"C语言程序"、"C++程序" |

### 5.3.2 高级检索（FR-411 ~ FR-420）

#### FR-411 多字段组合

```
检索表达式：
  (书名包含 "数据结构") AND (作者包含 "严") AND (分类 = "计算机")
  AND (出版年份 >= 2020) AND (价格 <= 100)
```

#### FR-412 逻辑运算符

| 运算符 | 优先级 | 说明 |
|--------|--------|------|
| `AND` | 2 | 同时满足 |
| `OR` | 3 | 满足任一 |
| `NOT` | 1 | 排除条件 |
| `()` | 最高 | 分组优先 |

#### FR-413 范围筛选

| 字段 | 范围类型 | 示例 |
|------|---------|------|
| 出版年份 | 闭区间 | `2020..2022` |
| 价格 | 比较运算 | `< 50`, `>= 100` |
| 借阅次数 | 比较运算 | `> 100` |

### 5.3.4 排序与筛选（FR-431 ~ FR-440）

#### FR-431 排序选项

| 排序方式 | 适用场景 | 算法 |
|---------|---------|------|
| 相关性 | 关键词检索后默认 | 匹配字段数+匹配位置权重 |
| 借阅热度 | 发现热门图书 | borrow_count 降序 |
| 出版年份 | 找最新资料 | year 降序 |
| 价格 | 预算有限时 | price 升序/降序 |
| 书名 | 按字母顺序 | title 字典序 |

#### FR-432 相关性评分算法（简化）

```
score = 书名匹配(权重3) + 作者匹配(权重2) + ISBN匹配(权重1)
      + 分类匹配(权重1) + 出版社匹配(权重0.5)

若书名开头匹配，额外+5分
```

## 5.4 非功能需求

### 5.4.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 单次检索响应 | ≤1秒（10000条记录） |
| 模糊检索响应 | ≤2秒 |
| 分类浏览加载 | ≤0.5秒 |
| 结果分页 | 每页20条，翻页≤0.3秒 |

### 5.4.2 检索质量

- 精确检索准确率：100%
- 模糊检索召回率：≥90%（关键词在书名/作者中）
- 排序合理性：Top10结果包含目标图书的概率≥80%
