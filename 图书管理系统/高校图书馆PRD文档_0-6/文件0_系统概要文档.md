# 高校图书馆智能借阅管理系统

## 产品概要

### 0.1 产品背景
某高校图书馆需要对图书借阅、读者管理、逾期处理、罚款计算、借阅统计等进行高效管理。系统需支持多类型读者（本科生、研究生、教师、职工）的不同借阅权限，采用纯C语言CLI环境实现，使用简单数据结构（数组、链表、字典）而非数据库。

### 0.2 产品目标
- 实现读者分类精细化管理，支持四种身份差异化权限
- 提供完整的图书借还流程，支持多副本库存管理
- 自动化逾期检测与罚金计算，支持催还与缴费
- 提供多维度统计报表与借阅排行榜
- 实现智能检索系统，支持组合查询与热门推荐
- 保障数据安全，实现持久化存储与备份恢复

### 0.3 核心约束条件
| 约束项 | 说明 |
|--------|------|
| 开发语言 | C语言 |
| 运行环境 | CLI命令行界面 |
| 数据结构 | 数组、链表、哈希索引（禁止数据库） |
| 持久化 | 二进制文件序列化 |
| 并发 | 单用户操作（预留扩展接口） |
| 数据规模 | 图书≤20000种，读者≤10000人，记录≤50000条 |

### 0.7 关键术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 读者类型 | ReaderType | 本科生/研究生/教师/职工的枚举分类 |
| 借阅权限 | BorrowLimit | 包含最大可借数、借阅天数、续借次数 |
| 借阅记录 | BorrowRecord | 单次借还操作的完整跟踪信息 |
| 逾期 | Overdue | 超过应还日期未归还的状态 |
| 罚金 | Fine | 逾期产生的费用，按天计算 |
| 图书热度 | BorrowCount | 图书累计被借阅次数，用于排行榜 |
| 分类索引 | CategoryIndex | 加速分类检索的内存数据结构 |
| 宽限期 | GracePeriod | 到期后不计逾期的缓冲天数 |
