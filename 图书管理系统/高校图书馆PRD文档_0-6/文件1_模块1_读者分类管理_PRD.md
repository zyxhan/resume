# 模块1：读者分类管理 PRD

## 1.1 需求概述

### 1.1.1 背景
高校图书馆服务多类读者群体（本科生、研究生、教师、职工），不同群体在学术研究深度、在校时长、教学需求等方面存在显著差异，需通过差异化借阅权限实现资源合理分配。

### 1.1.2 目标
- 建立可扩展的读者分类管理体系
- 实现基于身份的差异化借阅权限自动配置
- 管理读者账户全生命周期（注册→认证→冻结→注销）
- 支持院系/部门维度的读者分布统计
- 实现权限变更的实时生效与历史追溯

### 1.1.3 范围

| 包含 | 不包含 |
|------|--------|
| 读者注册与身份认证 | 与校园一卡通系统的物理对接 |
| 四类读者权限模型 | 人脸识别等生物特征认证 |
| 账户状态管理（正常/冻结） | 读者信用评分体系（二期规划） |
| 院系部门管理 | 多校区跨馆通借通还 |

## 1.3 详细功能需求

### 1.3.1 读者类型体系（FR-001 ~ FR-010）

#### FR-001 读者类型枚举定义

| 类型编码 | 类型名称 | 标识符 | 适用人群 |
|---------|---------|--------|---------|
| 0 | 本科生 | `READER_UNDERGRAD` | 全日制本科在读学生 |
| 1 | 研究生 | `READER_GRADUATE` | 硕士/博士研究生 |
| 2 | 教师 | `READER_TEACHER` | 专任教师、教辅人员 |
| 3 | 职工 | `READER_STAFF` | 行政、后勤等非教学人员 |

**数据结构**：
```c
typedef enum {
    READER_UNDERGRAD = 0,
    READER_GRADUATE = 1,
    READER_TEACHER = 2,
    READER_STAFF = 3
} ReaderType;
```

#### FR-002 默认权限矩阵

| 读者类型 | 最大可借数 | 借阅期限 | 续借次数 | 续借期限 | 预约权限 |
|---------|-----------|---------|---------|---------|---------|
| 本科生 | 5本 | 30天 | 2次 | 30天 | 允许2本 |
| 研究生 | 8本 | 60天 | 2次 | 60天 | 允许3本 |
| 教师 | 15本 | 90天 | 3次 | 90天 | 允许5本 |
| 职工 | 10本 | 60天 | 2次 | 60天 | 允许3本 |

**配置接口**：
```c
void get_type_limits(ReaderType type, int *max_books, int *max_days, int *max_renew);
```

#### FR-003 权限继承规则

- **升学自动升级**：本科生→研究生，系统检测学号前缀变更，自动提升权限
- **毕业自动降级**：研究生毕业后转为校友类型（权限：2本/30天/0续借）
- **职称变动**：讲师→副教授→教授，教师权限保持不变（统一按教师类型处理）

#### FR-004 特殊权限申请（教师推荐）

- 教师可为其指导研究生申请临时扩容（+2本/次）
- 有效期：当前学期
- 需教师工号验证 + 系统管理员审批

### 1.3.2 读者注册管理（FR-011 ~ FR-020）

#### FR-011 自助注册流程

```
开始
  │
  ▼
输入基本信息（姓名、学号/工号、电话、邮箱）
  │
  ▼
选择读者类型（下拉枚举）
  │
  ▼
系统自动验证学号/工号格式
  │
  ├─ 格式合法 ──► 设置密码 ──► 选择院系 ──► 提交注册
  │                                    │
  │                                    ▼
  │                              系统生成读者ID
  │                                    │
  │                                    ▼
  │                              发送确认邮件（模拟）
  │                                    │
  └─ 格式非法 ──► 提示错误，返回重输 ◄──┘
```

**输入校验规则**：

| 字段 | 校验规则 | 错误提示 |
|-----|---------|---------|
| 姓名 | 2-20个汉字或英文 | "姓名长度需在2-20个字符之间" |
| 学号/工号 | 本科生：`20\d{8}`<br>研究生：`20\d{10}`<br>教师：`T\d{6}`<br>职工：`E\d{6}` | "学号/工号格式不符合该类型规范" |
| 手机号 | `1[3-9]\d{9}` | "请输入有效的11位手机号码" |
| 邮箱 | 含`@`且域名合法 | "邮箱格式不正确" |
| 密码 | 6-20位，含字母+数字 | "密码需6-20位且包含字母和数字" |

#### FR-012 管理员批量导入

- 支持 CSV 格式批量导入（模拟实现：按固定格式文本输入）
- 字段顺序：`学号,姓名,类型,院系,电话,邮箱`
- 导入前校验：学号唯一性、类型合法性
- 冲突处理：跳过重复学号，生成错误报告

#### FR-013 身份验证规则

- 注册时学号/工号与类型必须匹配（如 `T` 开头必须是教师）
- 同一学号/工号全局唯一，不可重复注册
- 邮箱唯一性校验（用于密码找回）

### 1.3.3 账户状态管理（FR-021 ~ FR-030）

#### FR-021 状态机定义

```
                    ┌─────────────┐
                    │   待审核    │◄──── 批量导入后
                    │  (PENDING)  │
                    └──────┬──────┘
                           │ 管理员审核通过
                           ▼
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   已注销    │◄──── │   正常    │────► │   已冻结    │
│ (INACTIVE)  │ 毕业 │ (ACTIVE)    │ 违规 │ (SUSPENDED) │
└─────────────┘      └──────┬──────┘      └──────┬──────┘
                            │                      │
                            │ 欠费缴清             │ 管理员解冻
                            │◄─────────────────────┘
                            │
                            ▼
                     ┌─────────────┐
                     │   已过期    │
                     │  (EXPIRED)  │◄──── 借阅证到期
                     └─────────────┘
```

#### FR-022 状态属性

| 状态 | 可借阅 | 可预约 | 可续借 | 可登录 | 可缴费 |
|-----|--------|--------|--------|--------|--------|
| 正常 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 冻结 | ✗ | ✗ | ✗ | ✓ | ✓ |
| 过期 | ✗ | ✗ | ✗ | ✓ | ✓ |
| 待审核 | ✗ | ✗ | ✗ | ✗ | ✗ |
| 已注销 | ✗ | ✗ | ✗ | ✗ | ✗ |

#### FR-023 冻结触发条件（自动）

| 触发条件 | 阈值 | 冻结范围 |
|---------|------|---------|
| 逾期未还图书 | ≥3本 | 暂停借阅 |
| 累计逾期天数 | ≥90天 | 暂停借阅 |
| 欠费金额 | ≥50元 | 暂停借阅 |
| 图书损坏未赔偿 | ≥1本 | 暂停借阅 |
| 恶意占座/违规 | 管理员认定 | 全功能冻结 |

#### FR-024 解冻流程

1. 读者缴清欠费/归还图书
2. 系统检测条件满足，自动标记"可解冻"
3. 管理员在后台确认解冻
4. 发送解冻通知（系统内消息）

### 1.3.4 读者信息维护（FR-031 ~ FR-040）

#### FR-031 个人信息修改范围

| 字段 | 读者自助修改 | 管理员修改 | 修改后生效 |
|-----|-------------|-----------|-----------|
| 姓名 | ✗ | ✓ | 即时 |
| 电话 | ✓ | ✓ | 即时 |
| 邮箱 | ✓（需验证） | ✓ | 验证后 |
| 密码 | ✓ | ✓ | 即时 |
| 院系 | ✗ | ✓ | 即时 |
| 类型 | ✗ | ✓ | 即时（需校验当前借阅） |

#### FR-032 类型变更规则

- **本科生→研究生**：需验证新学号，原借阅图书按新期限重新计算
- **研究生→教师**：保留原有借阅，新增权限即时生效
- **教师→退休**：自动转为校友类型，已借图书按原期限到期

**变更限制**：当前借阅数 > 新类型最大可借数时，禁止变更，需先归还至合规

### 1.3.6 数据持久化（FR-051 ~ FR-060）

#### FR-051 读者数据结构

```c
typedef struct {
    int id;                     /* 读者编号（系统生成，8位） */
    char name[50];              /* 姓名 */
    char student_id[20];        /* 学号/工号（唯一标识） */
    char phone[15];             /* 手机号 */
    char email[50];             /* 邮箱 */
    char password[20];          /* 密码（MD5哈希，此处简化存储） */
    char department[40];        /* 院系/部门 */
    ReaderType type;            /* 读者类型 */
    int max_books;              /* 最大可借数（冗余存储，便于查询） */
    int max_days;               /* 最大借阅天数（冗余存储） */
    int borrowed;               /* 当前已借数（运行时计算） */
    float fine_balance;         /* 欠费余额 */
    int is_active;              /* 账户状态：1正常, 0冻结 */
    time_t register_time;       /* 注册时间戳 */
    time_t expire_time;         /* 借阅证到期时间 */
    int violation_count;        /* 违规次数（累计） */
} Reader;
```

#### FR-052 序列化格式

- 文件：`readers.dat`
- 格式：二进制顺序存储
- 结构：`[int:count][Reader:reader1][Reader:reader2]...`
- 索引：内存中按学号建立哈希索引（加载时构建）

#### FR-053 自动备份策略

| 触发条件 | 备份文件命名 | 保留数量 |
|---------|------------|---------|
| 每日首次修改 | `readers_YYYYMMDD.bak` | 最近7天 |
| 批量导入后 | `readers_YYYYMMDD_HHMM.bak` | 最近3次 |
| 手动触发 | `readers_manual_YYYYMMDD_HHMM.bak` | 最近10次 |

## 1.4 非功能需求

### 1.4.1 性能需求

| 指标 | 目标值 | 测试方法 |
|-----|--------|---------|
| 读者注册响应时间 | ≤2秒 | 单用户注册计时 |
| 读者检索响应时间 | ≤1秒（10000条记录） | 模糊查询计时 |
| 批量导入处理速度 | ≥100条/秒 | 1000条CSV导入计时 |
| 内存占用 | ≤50MB（10000读者） | 进程内存监控 |

### 1.4.2 可靠性需求

- 数据写入后即时 `fflush`，降低崩溃丢失风险
- 每次修改自动触发备份，保留最近3个版本
- 启动时自动校验数据完整性，损坏时尝试恢复备份

### 1.4.3 安全需求

| 需求 | 实现方式 |
|-----|---------|
| 密码安全 | 存储MD5哈希值（简化实现），传输不显示明文 |
| 权限隔离 | 读者只能查看/修改自己的信息 |
| 操作审计 | 关键操作（冻结、类型变更）记录日志（内存数组） |
| 防暴力破解 | 连续5次密码错误锁定账户15分钟 |

## 1.7 验收标准

| 测试项 | 测试步骤 | 预期结果 |
|-------|---------|---------|
| 本科生注册 | 输入合法信息，选择类型0 | 生成ID，权限5/30 |
| 研究生注册 | 输入学号2022开头，选类型1 | 生成ID，权限8/60 |
| 学号格式校验 | 输入本科生学号2022开头 | 提示"学号格式不符合本科生规范" |
| 重复注册 | 使用已注册学号再次注册 | 提示"学号已存在" |
| 冻结触发 | 模拟欠费50元后尝试借阅 | 提示"账户已冻结" |
| 类型升级 | 本科生借阅3本后升级为研究生 | 保留3本，新增额度5本 |
| 类型变更限制 | 当前借阅8本，变更为本科生 | 提示"请先归还至5本以下" |
