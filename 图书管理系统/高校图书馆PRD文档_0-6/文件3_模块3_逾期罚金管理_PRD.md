# 模块3：逾期罚金管理 PRD

## 3.1 需求概述

### 3.1.1 背景
逾期管理是保障图书流通效率、维护公平借阅秩序的关键环节。需要自动化检测、计算罚金，并提供催还和缴费渠道。

### 3.1.2 目标
- 实现逾期状态的自动检测与标记
- 罚金计算透明、可追溯
- 提供多渠道催还通知
- 支持罚金缴纳与财务对账

### 3.1.3 范围

| 包含 | 不包含 |
|------|--------|
| 逾期自动检测 | 与财务系统对接 |
| 罚金计算与累计 | 在线支付接口 |
| 催还通知生成 | 短信网关对接 |
| 罚金缴纳记录 | 发票开具 |

## 3.3 详细功能需求

### 3.3.1 逾期检测引擎（FR-201 ~ FR-210）

#### FR-201 检测触发方式

| 触发方式 | 频率 | 场景 |
|---------|------|------|
| 定时扫描 | 每日凌晨2:00 | 全量逾期检测 |
| 实时检测 | 每次操作时 | 借阅/查询时检查 |
| 手动触发 | 管理员操作 | 即时生成逾期清单 |

#### FR-202 逾期状态标记

```c
/* 记录状态转换 */
RECORD_BORROWING ──► RECORD_OVERDUE  (当 current_time > due_time)
RECORD_RENEWED   ──► RECORD_OVERDUE  (当 current_time > due_time)
```

**标记规则**：
- 精确到天（忽略时分秒）
- 到期日当天不算逾期（宽限期）
- 逾期次日开始计算

#### FR-203 宽限期设置

| 读者类型 | 宽限期 | 说明 |
|---------|--------|------|
| 本科生 | 1天 | 到期次日开始计逾期 |
| 研究生 | 2天 | 考虑科研周期 |
| 教师 | 3天 | 考虑教学任务 |
| 职工 | 2天 | - |

### 3.3.2 罚金计算系统（FR-211 ~ FR-220）

#### FR-211 阶梯费率模型（完整版）

| 逾期天数 | 费率 | 日上限 | 累计上限 |
|---------|------|--------|---------|
| 1-7天 | 0.5元/天 | 0.5元 | 3.5元 |
| 8-30天 | 0.8元/天 | 0.8元 | 22.1元 |
| 31-60天 | 1.0元/天 | 1.0元 | 52.1元 |
| 61-90天 | 1.5元/天 | 1.5元 | 97.1元 |
| >90天 | 图书原价 | - | 图书原价 |

#### FR-212 简化费率模型（当前版本实现）

```
罚金 = 逾期天数 × 0.5元/天
无上限（或设置软上限100元）
```

#### FR-213 罚金累计规则

- 读者级累计：所有未缴纳罚金累加到 `reader.fine_balance`
- 记录级存储：每条记录独立存储罚金金额
- 缴纳时优先冲抵最早产生的罚金（FIFO）

### 3.3.3 催还管理（FR-221 ~ FR-230）

#### FR-221 通知分级策略

| 级别 | 触发条件 | 通知内容 | 频率 |
|-----|---------|---------|------|
| 提醒 | 到期前3天 | 温馨提醒即将到期 | 1次 |
| 警告 | 逾期1-7天 | 已逾期，请尽快归还 | 每3天1次 |
| 催还 | 逾期8-30天 | 严重逾期，产生罚金 | 每周1次 |
| 通报 | 逾期31-60天 | 将通报院系/导师 | 每2周1次 |
| 冻结 | 逾期>60天 | 账户已冻结，请立即处理 | 1次 |

#### FR-222 通知渠道（模拟）

| 渠道 | 实现方式 | 优先级 |
|-----|---------|--------|
| 系统消息 | 登录时弹窗 | 最高 |
| 邮件 | 模拟发送（记录日志） | 高 |
| 短信 | 模拟发送（记录日志） | 中 |
| 电话 | 仅标记需人工拨打 | 低 |

### 3.3.4 罚金缴纳（FR-231 ~ FR-240）

#### FR-231 缴纳流程

```
查询欠费金额
    │
    ▼
输入缴纳金额
    │
    ├─ 金额 < 欠费 ──► 部分缴纳
    │                    │
    │                    ▼
    │              更新欠费余额
    │                    │
    └─ 金额 ≥ 欠费 ──► 全额缴纳
              │
              ▼
        计算找零
              │
              ▼
        更新账户状态（若欠费清零且其他条件满足，解冻）
              │
              ▼
        生成缴纳记录
```

#### FR-232 缴纳记录

```c
typedef struct {
    int payment_id;             /* 缴纳编号 */
    int reader_id;              /* 读者编号 */
    float amount;               /* 缴纳金额 */
    float applied_to_fine;      /* 实际冲抵罚金 */
    float change;               /* 找零 */
    time_t payment_time;        /* 缴纳时间 */
    char operator_id[20];      /* 操作员（自助为"SELF"） */
} PaymentRecord;
```

### 3.3.5 豁免与减免（FR-241 ~ FR-250）

#### FR-241 豁免情形

| 情形 | 证明材料 | 审批 | 豁免范围 |
|-----|---------|------|---------|
| 因病住院 | 医院证明 | 管理员 | 住院期间逾期 |
| 因公出差 | 单位证明 | 管理员 | 出差期间逾期 |
| 系统故障 | 系统日志 | 自动 | 故障期间全部 |
| 自然灾害 | 官方公告 | 管理员 | 影响期间 |
| 毕业离校 | 离校单 | 自动 | 毕业前30天内 |

#### FR-242 减免审批流程

```
读者申请 ──► 提交证明材料 ──► 管理员审核 ──► 通过/拒绝
                                    │
                                    ▼
                              通过: 生成减免记录
                              拒绝: 说明原因，读者可申诉
```

### 3.3.6 报表统计（FR-251 ~ FR-260）

#### FR-251 逾期率分析

```
逾期率 = 当前逾期人数 / 总活跃读者数 × 100%
分类逾期率 = 某类型逾期人数 / 该类型总人数 × 100%
```

#### FR-252 罚金收入统计

| 维度 | 统计项 |
|------|--------|
| 时间 | 日/周/月/年累计 |
| 类型 | 各读者类型缴纳占比 |
| 原因 | 逾期罚金/损坏赔偿/遗失赔偿 |
| 渠道 | 自助缴纳/管理员代收 |

## 3.4 非功能需求

### 3.4.1 准确性

- 罚金计算精确到分（0.01元）
- 逾期天数计算：到期日次日开始，归还日不计入
- 闰年2月29日按正常日期处理

### 3.4.2 时效性

- 逾期状态变更：T+1日生效（每日凌晨扫描）
- 实时查询时显示当前逾期状态（可能未同步最新扫描结果）

## 3.7 验收标准

| 测试项 | 步骤 | 预期结果 |
|-------|------|---------|
| 逾期检测 | 到期次日扫描 | 状态变更为OVERDUE |
| 罚金计算 | 逾期10天 | 罚金5.00元 |
| 部分缴纳 | 欠费10元，缴纳5元 | 余额5元，账户仍冻结 |
| 全额缴纳 | 欠费10元，缴纳10元 | 余额0元，账户解冻 |
| 超额缴纳 | 欠费10元，缴纳20元 | 找零10元，余额0元 |
| 豁免申请 | 提交住院证明 | 管理员审核通过后罚金清零 |

## 3.8 附录

### 3.8.1 术语表

| 术语 | 定义 |
|------|------|
| 宽限期 | 到期后不计逾期的缓冲天数 |
| 阶梯费率 | 随逾期天数增加而提高的罚金单价 |
| FIFO | First In First Out，先产生的罚金先冲抵 |
| 催还级别 | 根据逾期时长划分的通知紧急程度 |
